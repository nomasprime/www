This series is for software engineers and engineering teams who want to increase their competitive advantage by effectively integrating software engineering AI agents into their workflow.

A software engineering AI agent assists with development and accelerates delivery.

Unlike previous assistive technologies, like code completion, snippets, and generators, which are fairly rigid and limited in their capabilities, an AI agent is far more dynamic, context-aware, and capable of transforming human intent into deliverable software. It can:

- Interpret human intent
- Inspect codebases
- Plan changes
- Write code
- Run tests
- Debug failures
- Create pull requests

And much more…

In this series, I’ll explore LLMs, reimagine harness engineering, see software engineering through an agent’s eyes, and develop agents into indispensable engineering infrastructure.
