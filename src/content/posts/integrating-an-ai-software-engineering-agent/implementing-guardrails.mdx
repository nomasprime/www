## Intro

We can drastically improve the situation, before we bring agents into the mix simply by implementing existing best practices with the shaping analogy in mind.

Humans and agents are dependencies. If we double down on existing best practices we'll get the best of both worlds: humans will be more productive with and without AI; boost when AI's available.

## Picking A Programming Language

- Other criteria more important
- LLM training data
- Typed (or shaped) languages

## Leading By Example

- Moving from shaped languages to code examples
- General repository code
- Scaffolding
	- Formalised into more malleable shapes as snippets or templates
	- Generators take this a step further by encoding the above into mini shape factories
- Generative AI, the ultimate shape factory, is layered on top but the existing examples available pre-AI act as intermediary guardrails (guidance), instead of generating from scratch, AI can use our existing generators
- We'll come back to examples when we start talking about final guardrails

## Let's Play Tetris — Badly

Now have our existing shape we can go about producing our first shape to connect to it.

Using existing planning practices, eg, epics and stories, we can describe the shape we want to produce.

Iteratively, through prompting, we can refine that shape, we always want iterations low. When we don't get the right shape we want to understand why — mini retros.

We'll start doing retros everywhere.

## OK, Now Let's Start Winning

We actually introduced our first final guardrail when we picked a typed language — typechecking.

Looking at our first attempt at Tetris, we can add another guardrail — B/TDD (unit, integration, end-to-end). New behaviour, fixing a bug — all TDD.

Extend further with formatting and linting.
