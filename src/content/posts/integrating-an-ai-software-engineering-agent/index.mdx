This series is for software engineers and engineering teams who are interested in maintaining their competitive advantage by effectively integrating an AI software engineering agent into their workflow.

An AI software engineering agent can transform requirements into code. Among other things, it can:

- Understand goals
- Inspect codebases
- Plan changes
- Write code
- Run tests
- Debug failures
- Create pull requests

Unlike previous assistive technologies, like code completion, snippets, and generators, an AI agent is more dynamic and can operate with a greater degree of autonomy.

In this series, I’ll be exploring LLMs, reimagining harness engineering, workflows, and the engineering practices that make AI agents indispensable in real teams.
